## 2. Описание предметной области

Система предназначена для регистрации обращений клиентов в компанию, фиксирования их статуса и ответственных сотрудников. Пользователи могут добавлять новые обращения, отслеживать этапы обработки и анализировать типы обращений. Решаемые задачи включают снижение потерь обращений, ускорение реакции на запросы и повышение качества обслуживания клиентов.

## 3. Пользователи системы

| Роль пользователя | Описание функций                                        |
|-------------------|---------------------------------------------------------|
| Клиент            | Оставляет обращение через форму (в перспективе).       |
| Оператор          | Регистрирует обращения, обновляет статусы, добавляет комментарии. |
| Администратор     | Настраивает справочники типов обращений и пользователей. |

---

## 4. Основные сущности предметной области (данные)

| Сущность   | Описание                                      | Основные атрибуты                                  |
|-----------|-----------------------------------------------|----------------------------------------------------|
| Клиент    | Лицо, которое отправляет обращение.           | ФИО, контактные данные, организация                |
| Обращение | Запрос или жалоба от клиента.                 | Номер, дата, тема, описание, статус, приоритет     |
| Сотрудник | Сотрудник, обрабатывающий обращения клиентов. | ФИО, должность, контакт, список закреплённых обращений |

## 5. Пользовательские сценарии (Use Case)

| № | Сценарий                         | Краткое описание                                        |
